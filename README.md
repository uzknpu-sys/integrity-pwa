# Доброчесна Поліція — PWA

Прогресивний вебзастосунок (PWA) для Уповноваженого підрозділу з питань
запобігання корупції НПУ. Встановлюється на телефон, працює офлайн,
підтримує push-сповіщення.

## Склад

| Файл | Призначення |
|------|-------------|
| `index.html` | Головний екран застосунку |
| `resources.html` | Офлайн-довідник |
| `sw.js` | Service Worker (офлайн-кеш + push) |
| `manifest.webmanifest` | Маніфест PWA |
| `icons/` | Іконки застосунку |

## Розгортання на GitHub Pages

1. Створіть репозиторій (наприклад `integrity-pwa`).
2. Завантажте **всі** файли цієї теки у корінь репозиторію
   (`index.html`, `sw.js`, `manifest.webmanifest`, теку `icons/` тощо).
3. Settings → Pages → Source: `Deploy from a branch` → гілка `main`, тека `/ (root)`.
4. Через хвилину застосунок буде доступний за адресою
   `https://<ваш-логін>.github.io/integrity-pwa/`.

> Усі шляхи відносні (`./`), тож працює і в підтеці GitHub Pages,
> і на власному піддомені (`app.integrity-police.pp.ua`) без правок.

## Власний піддомен (необов'язково)

1. Settings → Pages → Custom domain → `app.integrity-police.pp.ua`.
2. У DNS додайте `CNAME` запис: `app` → `<ваш-логін>.github.io`.

## Push-сповіщення — що ще потрібно

Код у `sw.js` уже вміє **показувати** push. Щоб **надсилати** їх,
потрібен бекенд (це не можна зробити лише статичними файлами):

1. Згенерувати пару VAPID-ключів.
2. На клієнті підписатись через `pushManager.subscribe()` з публічним ключем.
3. Зберігати підписки та надсилати повідомлення зі свого сервера
   (Node `web-push`, або сервіс на кшталт OneSignal/Firebase Cloud Messaging).

Якщо хочете — підготую окремий модуль підписки + приклад бекенду.

## Перевірка

Відкрийте сайт у Chrome → DevTools → Lighthouse → категорія PWA.
Має бути «Installable» та зеленим пунктом офлайн-доступ.
