# ПОЧНИ ТУТ — оновлення застосунку + сповіщення

Ця інструкція саме під ваш випадок: застосунок уже на GitHub
(uzknpu-sys.github.io/integrity-pwa), але там стара версія без довідника
і без сповіщень. Зробимо два кроки по черзі.

═══════════════════════════════════════════
КРОК 1. ОНОВИТИ ЗАСТОСУНОК (спочатку без сповіщень)
═══════════════════════════════════════════

Мета: щоб з'явився повний довідник (40 задач) і кнопка сповіщень.
Цей крок працює навіть якщо Firebase ще не налаштований.

1. Зайдіть на github.com у свій репозиторій integrity-pwa.
2. Натисніть «Add file» → «Upload files».
3. Розпакуйте новий архів на комп'ютері та перетягніть у вікно
   браузера ВСІ ці файли (перезапишуть старі):
      • index.html
      • guide.html         ← новий повний довідник
      • sw.js
      • firebase-messaging-sw.js
      • manifest.webmanifest
      • теку icons (усі іконки)
   resources.html можна не заливати — вона більше не потрібна.
4. Унизу натисніть «Commit changes».
5. Зачекайте 1-2 хвилини.

Перевірка на телефоні:
6. Видаліть старий ярлик застосунку з головного екрана
   (утримати значок → Видалити).
7. Відкрийте https://uzknpu-sys.github.io/integrity-pwa/ у Safari.
8. Поділитися → «На початковий екран».
9. Відкрийте з нового ярлика.

✅ Якщо все вийшло: офлайн-картка тепер називається
   «Довідник стандартів доброчесності» (11 тем, 40 задач),
   а нижче з'явилася кнопка «🔔 Увімкнути сповіщення».

⚠️ Кнопка сповіщень поки що НЕ працюватиме, поки не зробите Крок 2.
   Але якщо кнопка З'ЯВИЛАСЯ — значить, оновлення пройшло успішно.

═══════════════════════════════════════════
КРОК 2. НАЛАШТУВАТИ СПОВІЩЕННЯ (Firebase з нуля)
═══════════════════════════════════════════

Робіть це ТІЛЬКИ після того, як Крок 1 вдався і кнопка з'явилася.

── 2.1. Створити проєкт Firebase ──
1. Відкрийте console.firebase.google.com, увійдіть Google-акаунтом.
2. «Create a project» → назва integrity-police → Analytics можна
   вимкнути → «Create project».

── 2.2. Зареєструвати вебзастосунок ──
3. На головній проєкту натисніть значок «</>» (Web).
4. Назва integrity-pwa → «Register app».
5. З'явиться код firebaseConfig = { ... } — НЕ закривайте вікно,
   звідси копіюватимемо. Виглядає так:
      apiKey: "AIza..."
      authDomain: "integrity-police.firebaseapp.com"
      projectId: "integrity-police"
      storageBucket: "integrity-police.appspot.com"
      messagingSenderId: "1234567890"
      appId: "1:1234567890:web:abc123"

── 2.3. Отримати VAPID-ключ ──
6. Шестерня ⚙ (вгорі) → «Project settings».
7. Вкладка «Cloud Messaging».
8. Розділ «Web Push certificates» → «Generate key pair».
9. З'явиться довгий рядок з літери «B...» — це VAPID-ключ, скопіюйте.

── 2.4. Вставити ключі у ДВА файли ──
Відкрийте файли текстовим редактором (Блокнот / Notepad++).

У файлі index.html знайдіть унизу блок із «ЗАМІНІТЬ_»:
      apiKey: "ЗАМІНІТЬ_apiKey",
      ...
      const VAPID_KEY = "ЗАМІНІТЬ_публічний_VAPID_ключ";
   Замініть кожне значення на своє з кроків 2.2 і 2.3.

У файлі firebase-messaging-sw.js — той самий блок «ЗАМІНІТЬ_».
   Впишіть ТІ САМІ значення з кроку 2.2 (VAPID сюди не треба).

   ⚠️ messagingSenderId та appId мають збігатися в обох файлах.
   ⚠️ Лапки "..." не видаляйте.

── 2.5. Перезалити на GitHub ──
10. github.com → репозиторій → Add file → Upload files →
    перетягніть оновлені index.html і firebase-messaging-sw.js →
    Commit changes.

── 2.6. Підключити сповіщення на телефоні ──
11. Видаліть старий ярлик, відкрийте сайт у Safari заново,
    додайте на головний екран, відкрийте з ярлика.
12. Натисніть «🔔 Увімкнути сповіщення» → «Дозволити».
13. Має з'явитися «✓ Сповіщення увімкнено».

── 2.7. Надіслати тестову новину ──
14. Firebase Console → ліве меню «Messaging» →
    «Create your first campaign» → «Firebase Notification messages».
15. Впишіть заголовок і текст → «Send test message» або
    «Review» → «Publish».

✅ На телефоні має з'явитися БАНЕР зверху екрана.

═══════════════════════════════════════════
ВАЖЛИВО ПРО iPhone (щоб не шукати дарма)
═══════════════════════════════════════════

• Сповіщення працюють ТІЛЬКИ при відкритті з ярлика на головному
  екрані, НЕ з Safari.
• Потрібен iOS 16.4 або новіший.
• «Циферка на ярлику як у WhatsApp» на iPhone для таких застосунків
  майже не працює — НЕ орієнтуйтесь на неї. Головна ознака успіху —
  це БАНЕР-сповіщення зверху і запис у Центрі сповіщень.
• Після дозволу перевірте: Налаштування iPhone → Доброчесна Поліція →
  Сповіщення → переконайтесь, що увімкнено.

═══════════════════════════════════════════
ЯКЩО КНОПКА НЕ З'ЯВИЛАСЯ ПІСЛЯ КРОКУ 1
═══════════════════════════════════════════

Найчастіша причина — телефон показує стару версію з кешу.
Рішення: видаліть ярлик, у Safari закрийте всі вкладки сайту,
зайдіть заново і додайте на екран ще раз. iPhone інколи тримає
старий кеш — повторне додавання його скидає.
